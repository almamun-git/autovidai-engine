How to contribute

- File an issue: If you found a bug or have a feature request, open an issue with a short reproducible description and suggested outcome.
- Pick an issue: Look for issues labeled `good first issue` or `help wanted`.
- Create a branch: Use descriptive branch names such as `fix/typo-readme` or `feat/stage3-retry`.
- Make changes: Keep changes focused and small. Add tests where appropriate.
- Commit messages: Use clear, present-tense messages. Example: `feat(frontend): add prompt editor`.
- Open a pull request: Target the `main` branch and describe the problem you solved and the approach taken.

Local development

- Python backend: create a virtual environment and install dependencies from `backend/requirements.txt`.
- Frontend: run `npm install` in `frontend` and `npm run dev` to start.

Testing

- Add tests for new functionality where applicable and run existing test suites before opening a PR.

Code style

- Follow the existing project formatting and linting rules. Keep changes consistent with surrounding code.

Review and merge

- PRs will be reviewed and merged once they meet quality and testing expectations. Maintainters may request changes.

Security

- For security issues, please follow `SECURITY.md` to report privately.

Acknowledgements

- Thanks to everyone who contributes. If you want recognition in release notes, mention it in your PR.
